## Mental model for the next reader

**The theory.** The Riemann Hypothesis says all non-trivial zeros of ζ have Re(ρ) = ½. Search has verified this for ~10¹³ zeros (Odlyzko, Platt–Trudgian) but a rogue zero could hide at arbitrarily high height. The project's conjecture: under a multiplicative gauge T_c(s) = s · τ^k (τ = 2π), the RH predicate is **transport-invariant**. If true, the bounded-view corollary kicks in: any high zero has an image under T_c^k that lands in a finite computable window. Search becomes finite. The project is not a proof; it's evidence-gathering for the transport claim.

**Four obligations on the critical path:** OBL_COORD_RECONSTRUCTION_COVARIANCE (EXP_1 candidate witness), OBL_ZERO_SCALING_EQUIVALENCE (no witness in current architecture), OBL_BETA_INVARIANCE (EXP_6), OBL_EXACT_RH_TRANSPORT (load-bearing, no empirical witness possible — needs a formal proof). Plus controls (EXP_1B, EXP_3) and pathfinders (EXP_4, EXP_5).

**Program 1 vs. Program 2.** Program 1 is the direct invariance attack. Program 2 is "contradiction by detectability" (rogue zeros amplify under deep zoom). Per Decision Log #2, Program 2 must NOT contribute to the theorem verdict. EXP_2/EXP_2B/EXP_7 are Program 2.

**The presentation gap that motivated the original work.** The dashboard's `overall: FAIL` and `REFUTES` stage labels read as if the theory was being refuted. But the obligation block (the actual theorem-state) hadn't moved across 23 runs. Threshold misses were being rendered with theorem-level vocabulary. The analyzer's deterministic decision table fixed the presentation; the kernel rebuilds fix the underlying numbers it interprets.

## Gotchas

- **Don't run `python -m analyzer` and expect new insights without re-running the engine first.** The artifact is stale.
- **Don't trust EXP_6's β̂ ≈ 0.57** until the rebuilt test has run. The number was generated against a flawed test design.
- **Don't sign off `WITNESS_MAP_REVIEW_STATUS = SIGNED_OFF`** without first deciding the EXP_1 / EXP_1C promotion question (see [obligation_movement_analysis.md §4](obligation_movement_analysis.md)). With the current witness map, no obligation can ever reach `WITNESSED`.
- **Bridge tests need the experimental flag:** `NODE_OPTIONS=--experimental-vm-modules npx jest __tests__/mcp-bridge-fallback.test.ts`. Other tests don't.
- **Pytest must be installed** for `tests/test_proof_program.py` to run. The new analyzer tests use bare asserts and work either way.
- **The dashboard reads `summary.stage_verdicts`.** After a fresh engine run, `brittleness` will not be in this dict — it'll be in `summary.program_2_summary`. This is intentional. The dashboard may need a small UI update to render `program_2_summary` explicitly. If you don't update it, the dashboard simply won't show the brittleness card — which is honest, just visually different from before.
- **Don't introduce new Unicode in `analyzer/render.py` outputs without testing on Windows cp1252.** Stdout is reconfigured to UTF-8 in `__main__.py`, but stray hard-Unicode in the output template can still bite. The render uses ASCII-only arrows now (`->`).
